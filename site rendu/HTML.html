Qu'est-ce que le HTML?

"HyperText Markup Language" (ou "langage de balisage de l'hypertexte" pour les anglophobes) est un langage informatique qui sert à indiquer à votre navigateur comment afficher une page web, dont la syntaxe vient du Standard Generalized Markup Language (SGML).. Entre autre chose, le développeur peut choisir quoi mettre sur sa page grâce au HTML, du texte le plus simpliste à la vidéo la plus ambitieuse. Il sert de squelette à tous les sites que vous pourriez trouver sur Internet. HTML est une des trois inventions à la base du World Wide Web, avec le Hypertext Transfer Protocol (HTTP) et les adresses web (URL).

HTML est une des trois inventions à la base du World Wide Web (en 1989), avec le Hypertext Transfer Protocol (HTTP) et les adresses web (URL). HTML a été inventé pour permettre d'écrire des documents hypertextuels liant les différentes ressources d’Internet avec des hyperliens. Aujourd’hui, ces documents sont appelés « page web ». C'est le le World Wide Web Consortium (W3C) qui met en place les différentes versions du langage qui est aujourd'hui au HTML 5.

Comment ça fonctionne ?

Le HTML est un langage interprété par le navigateur qui sert de squelette à une page web. Le navigateur va « lire » les instructions, que l’on appelle « balises », pour afficher une page lisible par un être humain. Les balises sont donc des instructions pour notre navigateur.

Les balises sont toutes composées de chevrons ouvrants "<" et fermants ">". Il existe 2 types de balises (par paire, et auto-fermante). Si des balises en contiennent d’autres, les balises contenu sont appelées « enfants » et les balises conteneurs sont les « parents »

Les balises par paire contiennent du texte, encadré par deux balises (voir code ci-dessous). Elles indiquent au navigateur le début et la fin de notre texte, ainsi que sa nature. La balise fermante contient toujours un slash « / » après le chevron.

Les balises orphelines ne contiennent rien, elle sont seules. Ces balises permettent le plus souvent d’insérer un élément dans notre page. Par convention, il est préférable de mettre un slash « / » à la fin de notre balise, même si ce n’est pas obligatoire.
